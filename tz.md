Собеседование
Тестовое задание: сервис автообъявлений

Разработать систему сбора автомобильных объявлений с API, админ-панелью и Telegram-ботом.

1. Backend & Скрапер

Воркер периодически собирает объявления с carsensor.net. Обязательные поля: марка, модель, год, цена, цвет, ссылка. Логика upsert: обновлять существующие записи при изменениях, добавлять новые. Базовая retry-логика при сетевых ошибках.

БД PostgreSQL/MySQL. Миграции через Alembic, сидинг с дефолтным администратором. Эндпоинты:
- POST /api/login — авторизация, возврат JWT
- GET /api/cars — защищённый список автомобилей

2. Фронтенд (SPA)

React или Next.js, любые UI-библиотеки.
- /login — форма входа, сохранение JWT
- / — защищённый роут, таблица автомобилей из API

3. Telegram-бот

Принимает запросы в свободной форме («Найди красную BMW до 2 млн»). Через Function Calling любого LLM API извлекает параметры фильтрации, делает запрос в БД и возвращает читаемый ответ. Нет доступа к LLM — напишите, выдадим ключ.

4. Сдача

- Запуск одной командой: docker-compose up --build
- Публичный репозиторий, файл .env.example
- README: инструкция по запуску, логин/пароль админа, архитектурные решения
